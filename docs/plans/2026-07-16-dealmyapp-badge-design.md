# DealMyApp 徽章接入设计

日期：2026-07-16

## 目标

在 `https://previewship.com/try` 页面提供可由 DealMyApp 验证爬虫直接读取的反向链接，同时不干扰访客发布静态产物的核心流程。

## 已确认方案

仅在 `/try` 页面主内容之后增加轻量页脚，居中展示 DealMyApp 提供的浅色徽章。链接和图片地址使用官方片段，链接保持可跟随，不添加 `nofollow`、`sponsored` 或 `ugc`。

未选择以下方案：

- 同时展示在首页和 `/try`：能够增加曝光，但会重复放置目录徽章，超出本次验证需要。
- 仅展示在首页：改动最小，但 DealMyApp 当前验证地址是 `/try`，无法保证验证成功。

## 组件与布局

不新增通用组件。徽章只使用一次，直接放入 `GuestDeployPage` 的语义化 `footer`，避免为了单一元素增加抽象。页脚使用现有页面的颜色、边框和响应式宽度规则，徽章保持官方 `200 × 40` 比例。

## 数据与外部依赖

页面不新增状态、接口或埋点。服务端静态渲染直接输出 `<a>` 与 `<img>`，浏览器从 DealMyApp 加载 SVG；如果外部图片暂时不可用，图片替代文本仍说明链接用途，页面发布功能不受影响。

## 验证

1. 执行前端生产构建。
2. 检查生成的 `/try` HTML 包含目标 listing URL 和徽章 SVG URL。
3. 确认锚点没有 `nofollow`、`sponsored` 或 `ugc`。
4. 确认 TypeScript、SSG 和 SEO 校验均通过。
