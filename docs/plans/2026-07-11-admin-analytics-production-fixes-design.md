# 管理员统计页生产验证修正设计

## 目标

修复生产验证中发现的四类问题：私有路由深链 404、汇总 KPI 与 cohort 漏斗共用详情口径、诊断维度与漏斗不一致、订阅明细展示当前套餐而非历史购买套餐，以及公开 SSG 页面语言水合不一致。

## 方案选择

详情接口继续复用现有管理员权限、白名单筛选和分页模型，新增 `scope=summary|cohort`：

- `summary` 对应顶部全量经营 KPI 和趋势；
- `cohort` 对应严格逐层收敛的获客漏斗和 Checkout 漏斗；
- 旧客户端未传 `scope` 时默认 `summary`。

未选择拆分两套详情接口，因为会复制权限、分页和字段模型；未选择取消漏斗下钻，因为会重新失去管理员查看来龙去脉的能力。

## 数据口径

- 访问 KPI 详情使用与概览相同的唯一身份优先级，并按身份保留一条代表记录。
- 发布成功 KPI 详情按唯一发布者返回一条代表部署，不能按部署次数返回。
- 获客漏斗详情复用同一 cohort CTE，阶段依次为访问、提交、成功、注册或认领。
- 来源、地区、落地页和发布渠道均从 cohort 的代表事件或代表部署归因，维度合计与漏斗阶段保持一致。
- 激活、新增订阅和净收入继续使用业务事实明细。
- 新增订阅与收入的套餐优先取关联 Checkout 的 `target_plan` 和 `billing_cycle`，无法关联时才回退当前订阅状态。

## 前端交互

- KPI 和趋势打开 `summary` 详情。
- 获客漏斗、诊断维度和基于 cohort 的经营建议打开 `cohort` 详情。
- 失败原因使用全量失败事实；Checkout 漏斗继续使用同一 Session cohort。
- 抽屉继续保持脱敏字段、分页和移动端单列布局。

## 路由与水合

- 将 `/admin/analytics` 加入私有 noindex 路由 HTML 生成和 Vercel rewrite，支持直接访问、刷新和登录返回。
- 私有路由生成无预渲染应用壳，使用 `createRoot`。
- 对公开预渲染页面，在 `hydrateRoot` 前按照 URL 语言段切换 i18n，避免本地语言缓存与 SSG HTML 不同导致 React hydration mismatch。

## 验收

- 7 天生产样本中 KPI 访问 78 的详情为 78，发布成功 523 的详情为 523。
- 漏斗 68 → 11 → 9 → 2 的四个详情分别返回相同数量。
- “成功”阶段的来源、地区、落地页、发布渠道合计与 cohort 成功人数一致。
- 新增订阅历史套餐不再因后续取消显示为 `FREE · NONE`。
- `/admin/analytics` 直接访问和刷新返回应用页面，不再 404。
- 桌面、390px 移动端、分页、日期范围、无障碍与控制台错误检查通过。
