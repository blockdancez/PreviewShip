# PDF 全屏阅读 X 帖子与配图设计

## 目标

用一个真实、常见的文件分享场景介绍 PDF 全屏阅读能力。内容站在发送文件和接收文件的用户视角，不采用产品公告或广告口吻。

## 叙事角度

用户把报告、方案或设计稿发给同事、客户后，对方通常需要先下载文件，再寻找并打开。新能力把这段流程缩短为：收到链接、点击、直接在浏览器全屏阅读。

帖子只描述这个小但具体的体验改善，不使用“重磅上线”“立即体验”等促销表达，也不添加外部链接或强制互动话术。

## 视觉方向

采用自然的编辑工作台场景：聊天窗口里出现一个 PDF 链接，旁边浏览器已经以原生阅读器全屏显示文档。画面重点是“从分享链接到直接阅读”的连续体验。

- 不放 PreviewShip Logo、价格、按钮或宣传口号。
- 不生成大段可读 UI 文案，避免伪造具体产品界面。
- 使用克制的编辑插画或自然产品场景，适合 X 横向信息流。
- 画面保留清晰焦点，避免广告海报式构图。

## 交付物

- 一篇简短的中文 X 单帖。
- 一张横向配图，与帖子互补而不是重复帖子文字。
