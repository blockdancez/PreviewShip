# PreviewShip

PreviewShip 为非专业用户提供静态产物发布和固定预览链接，并通过增长经营数据理解用户从访问到持续使用和付费的过程。

## Language

**增长经营驾驶舱**：
仅供管理员使用的经营视图，依次回答业务趋势、转化阻塞原因和下一步行动，不等同于原始事件报表。
_Avoid_: 数据大屏、埋点列表、万能后台

**产品事件**：
用户或系统在产品旅程中发生的行为记录，可关联匿名访客、会话、获客、用户、项目和部署。
_Avoid_: 日志、点击流水

**业务事实**：
由用户、项目、部署、结账、订阅和支付状态直接证明的业务结果，其可信度高于客户端行为事件。
_Avoid_: 转化事件、埋点事实

**内部获客归因**：
PreviewShip 自有数据记录的首次落地页、来源站点、UTM、广告标识、地区和语言，用于解释用户如何进入产品。
_Avoid_: 全渠道归因、外部平台归因

**唯一用户旅程**：
登录前以获客标识识别同一访客，注册或认领后以用户标识识别同一用户，并通过注册或认领行为连接两个阶段。
_Avoid_: 事件漏斗、PV 漏斗

**激活用户**：
首次成功发布后七日内再次成功发布的已登录用户，代表用户不只完成一次试用，而是开始重复使用产品。
_Avoid_: 注册用户、首次发布用户

**经营建议**：
由明确指标阈值触发、附带对比基线和影响范围的管理员行动建议，每条建议都能追溯到对应数据。
_Avoid_: AI 建议、自动结论、运营文案

**经营结论**：
对一个统计周期内最重要趋势和主要转化阻塞的简明陈述，由已确认指标和经营建议共同支撑。
_Avoid_: 数据摘要、AI 总结

**数据链路完整度**：
能够从业务结果追溯到唯一用户旅程和内部获客归因的结果占比，用于标识分析结论的可信范围。
_Avoid_: 埋点成功率、数据准确率

**内部部署账号**：
由公司持有、同时供内部系统调用部署能力和授权人员登录维护且不参与产品用户增长经营的专用账号。
_Avoid_: 内部服务账号、公司用户、免费账号、平台管理员

**内部部署**：
由内部部署账号发起、用于公司系统发布静态产物且不进入产品用户增长经营分析的部署。
_Avoid_: 员工部署、免费部署、管理员部署

**共享内部项目**：
由内部部署账号下的项目名唯一标识、供所有同名内部调用共同更新同一固定预览链接的项目。
_Avoid_: API Key 私有项目、调用方私有项目

**内部固定预览链接**：
共享内部项目持续使用且不自动过期、始终展示其当前成功部署的稳定访问地址。
_Avoid_: 永久部署版本链接、临时预览链接

**内部回滚产物**：
每次成功内部部署持续保留至共享内部项目被删除、用于恢复该历史版本的原始静态产物。
_Avoid_: 部署记录、当前站点文件、临时上传文件

**当前内部版本**：
最近完成并成功激活、正在由内部固定预览链接展示的内部部署版本。
_Avoid_: 最后提交版本、最高版本号

**内部回滚**：
使用某个内部回滚产物创建一次新的内部部署、成功后将其内容恢复为当前内部版本的操作。
_Avoid_: 切换历史记录、修改旧部署、重新部署当前版本

**内部版本历史**：
共享内部项目中不受套餐数量窗口限制、始终可被维护人员和调用方查找的全部内部部署记录。
_Avoid_: 最近版本、回滚产物列表、构建日志

**内部项目访问**：
共享内部项目默认公开访问，并可由内部部署账号切换为不受普通套餐限制的密码访问。
_Avoid_: K8s 内网访问、平台管理员访问、默认私有访问

**内部账号安全邮件**：
仅为内部部署账号的身份验证和访问恢复发送、不包含部署生命周期或营销内容的必要邮件。
_Avoid_: 部署通知、到期提醒、订阅邮件、营销活动邮件

**内部部署可观测性**：
仅用于维护内部部署状态、错误、构建日志、处理耗时和系统健康，且不进入产品增长或经营分析的技术信息。
_Avoid_: 产品事件、经营指标、用户增长数据

**内部部署业务额度**：
内部部署账号不因会员套餐或商业权益受到项目数量、部署次数和累计上传量等使用额度限制。
_Avoid_: 无限资源、内部 Pro 套餐、免费额度

**部署安全边界**：
为保护共享部署能力免受异常文件、恶意内容或突发流量破坏而设置，且不用于会员商业分层的技术保护边界。
_Avoid_: 套餐额度、会员限制、资源无限

**内部共享部署凭证**：
由所有获授权公司系统共同使用、统一代表内部部署账号发起内部部署的单一 API Key。
_Avoid_: 调用方凭证、每服务 API Key、每环境 API Key、调用方身份

**内部共享部署凭证轮换**：
先撤销当前内部共享部署凭证、再创建新凭证，并接受两者之间内部系统无法发起部署的维护过程。
_Avoid_: 无中断轮换、双凭证过渡、自动轮换

**内部部署受理结果**：
确认内部部署请求已被接收并提供部署标识，但不承诺部署已经完成，也不直接提供预览链接的异步结果。
_Avoid_: 部署成功结果、预览结果、同步部署结果

**内部部署重试**：
在一次内部部署失败后由调用方重新提交产物并产生一条新内部部署记录的再次尝试。
_Avoid_: Worker 自动重试、恢复原部署、重复执行原任务

**内部部署失败摘要**：
向内部调用方说明失败类别和简要原因、但不包含完整构建日志的部署失败信息。
_Avoid_: 构建日志、服务端日志、错误堆栈

**内部账号控制台**：
供内部部署账号维护自身项目、部署、回滚、访问方式和共享部署凭证，并排除订阅、升级、商业额度和营销功能的登录后界面。
_Avoid_: 普通用户控制台、平台管理后台、内部运营后台

**内部维护登录**：
多个获授权维护人员使用同一内部部署账号登录，所有会话和操作统一归属于该账号而不区分具体人员。
_Avoid_: 团队成员、个人维护身份、操作人审计、平台管理员登录

**内部账号身份授予**：
由受控维护人员将一个已存在账号明确认定为内部部署账号，不能由账号自行申请或通过普通产品流程获得。
_Avoid_: 自助申请、邮箱自动识别、管理员角色、付费权益

**内部账号停用**：
通过封禁内部部署账号并撤销其内部共享部署凭证来终止登录和部署调用，同时永久保留其内部业务身份。
_Avoid_: 恢复普通账号、删除内部身份、套餐降级

**内部共享部署权限**：
内部共享部署凭证代表内部部署账号使用完整标准部署 API 的能力，包括部署、查询、回滚、访问方式修改和项目删除。
_Avoid_: 只读凭证、仅部署凭证、受限服务凭证

**内部安全封禁**：
因安全或滥用原因下线内部账号、项目或部署，并可删除相关在线内容与回滚产物的保护措施。
_Avoid_: 套餐限制、账号停用、普通部署失败

**内部项目名**：
由调用方提供、按原始字符串精确区分大小写和空白，并在同一内部部署账号内标识共享内部项目的名称。
_Avoid_: 规范化项目名、托管域名、大小写无关标识

**内部可部署产物**：
能够由浏览器直接打开的静态构建输出，或单个 HTML、Markdown、PDF 文件，不包含需要安装依赖或执行构建命令的前端源码。
_Avoid_: React 源码、Vue 源码、源码构建任务、运行时服务

**内部品牌豁免**：
内部部署和内部回滚不因套餐或营销目的向可发布内容注入 PreviewShip 品牌角标。
_Avoid_: Pro 权益、禁止产物转换、移除调用方品牌

**内部业务隔离**：
内部部署在身份、权益、统计、通知和生命周期上不与普通用户混淆，但不承诺独占部署执行或存储资源。
_Avoid_: 物理隔离、独立部署平台、资源零影响

## Relationships

- 一个**内部部署账号**拥有零个或多个**共享内部项目**
- 一次**内部部署**属于且只属于一个**共享内部项目**
- 同一**内部部署账号**下使用相同项目名的**内部部署**共同更新同一**共享内部项目**
- 一个**共享内部项目**拥有且只拥有一个**内部固定预览链接**，直到项目被删除或禁用前都不自动过期
- 每次成功**内部部署**产生一个不按时间或数量自动清理的**内部回滚产物**
- 多次**内部部署**并发时由最后完成并成功激活的部署成为**当前内部版本**，不按提交先后决定
- 每个被接受的内部部署请求都产生一次新的**内部部署**，不因请求标识或产物内容相同而合并
- 失败的**内部部署**保留失败记录，但不替换**当前内部版本**，内部固定预览链接继续展示上一个成功版本
- 一次**内部回滚**产生一条新的**内部部署**并关联其来源历史版本，不修改原历史部署记录
- **内部回滚**创建的新内部部署先产生**内部部署受理结果**，其成功、失败和预览链接通过同一部署状态查询获得
- **内部回滚**允许选择当前内部版本作为来源，此时仍创建并重新发布一条新的内部部署，而不是返回空操作
- **内部版本历史**包含成功、失败和处理中部署；只有仍有内部回滚产物的成功部署可以作为内部回滚来源
- **共享内部项目**遵循**内部项目访问**设置，公开与密码模式都不改变其内部业务身份
- **共享内部项目**可以由内部部署账号主动提交到 Showcase，公开展示不改变其内部业务身份，也不使内部部署进入增长经营分析
- **内部部署**不属于**唯一用户旅程**，也不构成增长经营使用的**业务事实**
- **内部部署账号**可以登录管理自己的项目、部署和 API Key，但只能访问自身资源，不获得平台级管理员权限
- **内部部署账号**只接收**内部账号安全邮件**，不接收部署、订阅、召回或营销邮件
- **内部部署**由账号身份而非请求网络来源判定，持有有效 API Key 的调用方可以通过标准公开 API 发起
- **内部部署**保留**内部部署可观测性**，但其技术信息不进入增长经营驾驶舱、激活计算或业务报表
- **内部部署**不受**内部部署业务额度**限制，但必须遵守适用于共享部署能力的**部署安全边界**
- **内部部署账号**通过一个**内部共享部署凭证**供所有获授权公司系统调用，部署归属只能追溯到内部部署账号而不能仅凭凭证区分具体调用系统
- **内部共享部署凭证轮换**期间不存在可用的内部共享部署凭证，调用方需要等待新凭证创建并完成配置
- 调用方收到**内部部署受理结果**后，通过部署标识查询内部部署状态，并仅在部署成功后取得预览链接
- 失败的内部部署不会自动再次执行；调用方需要发起**内部部署重试**，且原失败记录保持不变
- 失败的内部部署向调用方提供**内部部署失败摘要**；完整构建日志仅属于平台维护使用的内部部署可观测性
- **内部部署账号**登录后使用**内部账号控制台**，不进入普通用户的套餐购买、升级转化或营销交互
- **内部维护登录**可以同时存在多个会话，但只能审计到内部部署账号，不能追溯到具体维护人员
- **内部账号控制台**不展示订阅功能，但内部部署账号仍可通过既有商业接口直接创建订阅；订阅状态不改变其内部业务身份、内部部署业务额度或增长经营排除规则
- 一个账号只有经过**内部账号身份授予**才成为内部部署账号；该身份不由登录方式、API Key 或订阅状态决定
- **内部账号身份授予**作用于账号全部历史和未来数据；该账号既有项目、部署和产品事件同样不参与增长经营分析
- **内部部署账号**不再使用时执行**内部账号停用**，不恢复为普通账号，也不将其历史数据重新纳入增长经营分析
- **内部共享部署凭证**拥有**内部共享部署权限**；所有获授权公司系统共同承担项目删除、访问方式修改等高影响操作的风险
- **内部部署账号**及其项目、部署继续适用**内部安全封禁**；安全封禁优先于固定链接、当前版本和回滚产物的长期保留规则
- 只有完全相同的**内部项目名**才表示同一共享内部项目；大小写或空白不同的名称分别创建独立项目和固定预览链接
- 一次**内部部署**只接收**内部可部署产物**；React、Vue 等项目必须先由调用方构建为静态输出
- **内部部署**及**内部回滚**享有**内部品牌豁免**，不会生成角标访问和品牌获客事件
- **内部部署**与普通用户部署遵守**内部业务隔离**，但共享执行容量时可能相互影响排队时延
- **内部部署**不预留独立产物容量，也不设置专属安全水位；其长期保留产物可能影响普通用户可用的共享存储容量

## Flagged ambiguities

- “公司专属账号”既用于系统调用又允许人员登录维护，不是纯服务账号；统一称为**内部部署账号**
- “内部后管”统一表示维护**内部部署账号**自身的项目、部署和 API Key，不表示管理普通用户或整个平台
- “内部部署”中的“内部”表示公司业务用途，不表示只能从 K8s 或公司内网访问
- “部署去重”仅表示同一部署任务不会被 Worker 重复执行，不表示重复 HTTP 请求或相同产物会合并
- “不参与增长经营”不等于禁止公开展示；内部项目可以进入 Showcase，但相关内部部署仍不计入产品用户增长事实
